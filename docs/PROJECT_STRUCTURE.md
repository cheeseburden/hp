## Component Map — What Does What

### 🐍 Backend (FastAPI)

| File | Purpose | Real or Simulated |
|---|---|---|
| `main.py` | App startup — loads model, connects Kafka/ES/Vault, starts consumer | Real |
| `config.py` | Environment variables: `VAULT_ADDR`, `KAFKA_BOOTSTRAP_SERVERS`, etc. | Config |
| `schemas.py` | Pydantic models: `NetworkEvent`, `PredictionResult`, `PipelineStageResult` | Schemas |
| `inference.py` | Loads `.joblib` model, runs XGBoost + LightGBM ensemble predictions | **Real ML** |
| `threat_engine.py` | Orchestrates all 10 pipeline stages per event, updates global metrics | **Real** |
| `pipeline_stages.py` | Simulates stages 1-3 (Network, IDS, Beats), 6 (SOAR), 8-9 (Cred rotation/dist) | Simulated |
| `kafka_client.py` | Produces raw events to Kafka, consumes from `hpe-raw-events` topic | **Real** |
| `elastic_client.py` | Indexes audit logs and threats into Elasticsearch | **Real** |
| `vault_client.py` | Manages 200 user credentials in HashiCorp Vault, rotates on threat | **Real** |
| `ws_manager.py` | Broadcasts pipeline results to all connected WebSocket clients | Real |

### ⚡ Frontend (Vite + JS)

| File | Purpose |
|---|---|

## API Endpoints Quick Reference

### REST

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/health` | Full health check (model, kafka, ES, vault) |
| `GET` | `/api/metrics` | Pipeline metrics (threats, latency, attack types) |
| `GET` | `/api/pipeline/status` | All 10 pipeline stages with real health status |
| `POST` | `/api/predict` | Process a single `NetworkEvent` through the pipeline |
| `GET` | `/api/vault/credentials` | Latest rotated user's masked credentials |
| `GET` | `/api/vault/users` | **All 200 users** masked credentials (filter: `?role=`, `?region=`) |
| `GET` | `/api/vault/users/{user_id}` | Single user's masked credentials |
| `GET` | `/api/sample-events` | Test events count |

### WebSocket

| Endpoint | Description |
|---|---|
| `WS /ws/simulate` | Streams test events through the full pipeline in real-time |
